# PLANS

This repo uses the same execution shape it is trying to generate for other repos.

## Flow

```text
new work
  -> docs/exec-plans/active/
  -> execution and updates
  -> docs/exec-plans/completed/
```

## Rule

- active plans hold ongoing work
- completed plans hold full closure, not one-line summaries
- debt and backlog belong in `docs/exec-plans/tech-debt-tracker-v2.md`
